SQL Reporting Practice Project

# English Version

Project Overview
This repository contains my SQL practice work based on sample academic databases provided for training.

The main goal of this project is to improve my SQL query writing skills and strengthen my understanding of relational databases, table relationships, and reporting logic using MySQL.

Project Goal
The purpose of this project was to practise SQL in a practical way by working with multiple related databases and writing queries for data retrieval, joins, filtering, grouping, aggregation, reporting, and conditional logic.

Databases Used
The project is based on the following sample databases:

- sql_store
- sql_invoicing
- sql_hr
- sql_inventory

Tasks Performed
During this project, I worked on exploring the structure of multiple sample databases, understanding primary keys and foreign keys, analysing relationships between tables, and writing SQL queries using SELECT, JOIN, LEFT JOIN, GROUP BY, HAVING, ORDER BY, CASE, and SUM.

I also calculated totals, item counts, and order values, built reporting queries based on business-style data, and solved SQL exercises step by step.

Example Practice Topic
One of the exercises in this project focused on generating an order report from the sql_store database.

The report included the order ID, customer name, order date, order status, shipper name, total number of items, total order value, and shipping status.

This exercise helped me understand how to combine multiple tables and produce a meaningful summary report.

Skills Practised
This project helped me practise SQL query writing, relational database analysis, table joins, aggregation and grouping, conditional expressions in SQL, reporting, data summarisation, and structured problem solving.

Project Value
This is a beginner-to-intermediate SQL portfolio project.

Although it is not a full production system, it demonstrates practical SQL skills, an understanding of relational data, and the ability to solve reporting tasks using structured datasets.

# Deutsche Version

Projektübersicht
Dieses Repository enthält meine SQL-Übungen auf Grundlage akademischer Beispieldatenbanken, die zu Übungszwecken bereitgestellt wurden.

Das Hauptziel dieses Projekts ist es, meine Fähigkeiten im Schreiben von SQL-Abfragen zu verbessern und mein Verständnis für relationale Datenbanken, Tabellenbeziehungen und die Logik der Berichterstellung mit MySQL zu vertiefen.

Projektziel
Ziel dieses Projekts war es, SQL auf praktische Weise zu üben, indem ich mit mehreren zusammenhängenden Datenbanken gearbeitet und Abfragen für Datenabfrage, Tabellenverknüpfungen, Filterung, Gruppierung, Aggregation, Berichterstellung und bedingte Logik geschrieben habe.

Verwendete Datenbanken
Das Projekt basiert auf den folgenden Beispieldatenbanken:

- sql_store
- sql_invoicing
- sql_hr
- sql_inventory

Durchgeführte Aufgaben
Im Rahmen dieses Projekts habe ich die Struktur mehrerer Beispieldatenbanken untersucht, Primär- und Fremdschlüssel analysiert, Beziehungen zwischen Tabellen verstanden und SQL-Abfragen unter Verwendung von SELECT, JOIN, LEFT JOIN, GROUP BY, HAVING, ORDER BY, CASE und SUM geschrieben.

Außerdem habe ich Gesamtsummen, Stückzahlen und Bestellwerte berechnet, Reporting-Abfragen auf Grundlage geschäftsähnlicher Daten erstellt und SQL-Übungen schrittweise gelöst.

Beispiel eines Übungsthemas
Eine der Übungen in diesem Projekt konzentrierte sich auf die Erstellung eines Bestellberichts aus der Datenbank sql_store.

Der Bericht umfasste die Bestell-ID, den Kundennamen, das Bestelldatum, den Bestellstatus, den Namen des Versandunternehmens, die Gesamtanzahl der Artikel, den Gesamtwert der Bestellung und den Versandstatus.

Diese Übung hat mir geholfen zu verstehen, wie mehrere Tabellen miteinander kombiniert werden und wie daraus ein aussagekräftiger zusammenfassender Bericht erstellt werden kann.

Trainierte Fähigkeiten
Dieses Projekt hat mir geholfen, das Schreiben von SQL-Abfragen, die Analyse relationaler Datenbanken, Tabellenverknüpfungen, Aggregation und Gruppierung, bedingte Ausdrücke in SQL, Berichterstellung, Datenzusammenfassung sowie strukturiertes Problemlösen zu trainieren.

Wert des Projekts
Dies ist ein SQL-Portfolio-Projekt auf Anfänger- bis Mittelstufenniveau.

Obwohl es sich nicht um ein vollständiges Produktivsystem handelt, zeigt es praktische SQL-Kenntnisse, ein Verständnis für relationale Daten sowie die Fähigkeit, Reporting-Aufgaben mit strukturierten Datensätzen zu lösen.
